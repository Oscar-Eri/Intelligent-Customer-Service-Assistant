## 🎯 核心功能演示

### 功能1: 智能知识库检索

**用户提问**: "SW-2100智能手表的电池续航是多少?"

**系统处理流程**:
1. `intent_recognition_node` 识别为知识查询意图
2. `knowledge_retrieval_node` 检测到关键词"智能手表"
3. LLM 分析知识库摘要,推荐检索 `SW-2100-Flagship.md`
4. 检索到完整产品信息(500+字符)
5. 检索内容注入系统提示词
6. LLM 基于检索内容生成准确回答

**AI 回复示例**:
```
SW-2100旗舰智能手表的电池续航表现优秀:
- 典型使用: 72小时(3天)
- 省电模式: 14天
- 充电时间: 约2小时充满
- 电池容量: 500mAh,支持磁吸无线充电
```

### 功能2: 普通对话跳过 RAG

**用户提问**: "你好,今天心情怎么样?"

**系统处理**:
- 未检测到知识查询关键词
- 跳过 `knowledge_retrieval_node`
- 直接使用 LLM 生成自然回复
- **优势**: 节省 token,提高响应速度

### 功能3: 多文件对比查询

**用户提问**: "SW-2100和SW-1500有什么区别?"

**系统处理**:
- LLM 分析后推荐检索两个产品文件
- 同时检索 `SW-2100-Flagship.md` 和 `SW-1500-Sport.md`
- 整合两份信息进行对比分析

## 📊 技术亮点

### 1. 高内聚低耦合架构

```
┌─────────────────────────────────────┐
│     LangGraph Workflow              │
│  (业务逻辑层)                        │
└──────────────┬──────────────────────┘
               │ 调用
┌──────────────▼──────────────────────┐
│     RAG Module                      │
│  (独立模块,可复用)                   │
│  ├─ KnowledgeBaseManager            │
│  ├─ UnifiedLLMProvider              │
│  └─ SummaryGenerator                │
└──────────────┬──────────────────────┘
               │ 读取
┌──────────────▼──────────────────────┐
│     Knowledge Base Files            │
│  (Markdown 格式)                     │
└─────────────────────────────────────┘
```

**优势**:
- ✅ RAG 模块可独立测试和维护
- ✅ 可轻松移植到其他项目
- ✅ 替换 LLM Provider 无需修改业务代码
- ✅ 知识库格式变更不影响上层逻辑

### 2. 智能触发机制

```python
knowledge_keywords = [
    "产品", "规格", "参数", "技术", "文档",
    "如何", "怎么", "什么", "哪些"
]

# 只有包含关键词才触发 RAG
need_knowledge = any(kw in user_input for kw in knowledge_keywords)
```

**优势**:
- ✅ 避免不必要的 API 调用
- ✅ 降低 token 消耗
- ✅ 提高响应速度

### 3. 上下文注入优化

```python
if rag_used and rag_content:
    system_prompt = f"""
    ## 知识库检索结果
    {rag_content[:10000]}  # 限制长度
    
    ⚠️ 重要规则:
    1. 优先使用知识库检索结果回答问题
    2. 如果知识库中没有相关信息,请诚实说明
    """
```

**优势**:
- ✅ 明确的优先级指示
- ✅ Token 限制防止超出上下文
- ✅ 引导 LLM 正确使用检索内容
